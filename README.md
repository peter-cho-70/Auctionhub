# AuctionFlow Pro

다가구 등 **법원 경매 물건**을 단계별로 관리하고, 강의 노트·체크리스트·템플릿을 로컬에 저장하는 Next.js 앱입니다.

## 기능 개요

| 영역 | 설명 |
|------|------|
| **대시보드 / 물건** | 사건별 상태, 체크리스트, 입찰 회차, 메모. 기본: 물건 유형·**준공년도**, 면적·주차·위반, **건폐율·용적율·말소기준**(문자·템플릿 `{건폐율}` 등) |
| **임대세팅** | **앰버**: 핵심 입력, **입찰·낙찰 감정가 대비 %**(낙찰가는 도구 탭 입력 또는 판단 기록 실입찰가). 탭 전환 시 **기본 탭 미저장 초안**(감정가·면적 입력칸 등)까지 반영해 계산. 호실 표는 투자 **계산 결과** 바로 아래. 문자 미리보기도 동일 병합 물건 기준으로 `{감정가}`·`{감정가대비낙찰가}` 계산(기본 감정가 없으면 임대세팅 감정가 보조) |
| **프로세스** | 단계별 **체크리스트 템플릿** 편집 + **강의 노트**(단계별 문자열, 로컬 저장) |
| **공부하기 (`/study`)** | 요약본을 문서 미리보기 형태로 연속 열람, **원본 DOCX 다운로드** |
| **데이터** | 메시지 템플릿 등 부가 데이터 |

### 강의 요약본과 원본 파일

- **요약본**은 `src/lib/data/lecture-guide.ts`의 `LECTURE_BLOCKS_BY_STEP`에 단계(`CaseStatus`)별 블록으로 들어 있습니다.  
  프로세스 화면의 기본 강의 노트·공부하기 미리보기가 같은 내용을 사용합니다.
- 사용자가 프로세스에서 저장한 내용은 **`lectureGuideByStep`**(앱 데이터)에 덮어쓰며, 해당 단계에는 저장본이 우선합니다.
- **원본 DOCX**는 `public/lectures/originals/`에 두며, 메타데이터는 `src/lib/data/lecture-sources.ts`의 `LECTURE_ORIGINAL_DOCS`와 맞춥니다.  
  새 파일을 넣을 때는 폴더에 복사한 뒤 `lecture-sources.ts`에 항목을 추가하면 공부하기 페이지에 링크가 표시됩니다.

포함된 원본 예시(파일명):

- `곰물주_경매_강의노트_정리본.docx`
- `대출1강_1교시_정리본.docx`
- `명도_실무_매뉴얼_상세.docx`
- `명도1강_1교시_정리본.docx` · `명도1강_2교시_정리본.docx` · `명도1강_3교시_정리본.docx`

## 테마 (밝은 배경 / 어두운 배경)

- **밝게**: 밝은 회색 페이지 배경(`#f4f4f5`), 본문 대비용 표면색 변수 사용.
- **어둡게**: 짙은 배경(`#09090b`)과 대비되는 전경·테두리 변수 사용.
- **시스템**: OS의 `prefers-color-scheme`을 따릅니다.

설정은 헤더 오른쪽 토글에서 바꾸며, 브라우저 **`localStorage` 키 `auctionflow-theme`**에 `light` / `dark` / `system`으로 저장됩니다.  
첫 페인트 깜빡임을 줄이기 위해 `src/components/theme-script.tsx`가 페이지 로드 전에 `html`에 `dark` 클래스를 적용합니다.

스타일 진입점은 `src/app/globals.css`이며, Tailwind v4에서 **`@custom-variant dark`**로 `html.dark` 하위에 `dark:` 유틸이 동작하도록 했습니다.

## 최근 개선 내용

- **PDF로 물건 추가**
  - PDF 업로드 후 텍스트를 추출해 사건번호, 주소, 감정가, 최저가, 입찰일, 면적, 주차대수 등을 자동 입력합니다.
  - PDF 파일 선택 후 1초 뒤 자동으로 물건을 등록하고 상세 페이지로 이동하도록 개선했습니다.

- **원문/PDF 관리**
  - 물건 상세 화면에서 `원문/PDF` 탭을 가장 왼쪽에 배치했습니다.
  - PDF 원문과 구조화 JSON을 물건별 출처 문서로 저장해 이후 분석에 활용할 수 있도록 했습니다.
  - 원문/PDF 등록의 기본 문서 종류는 `건축물대장`으로 설정했습니다.

- **매각물건명세서 기반 세입자 보강**
  - 세입자 분석 화면에서 매각물건명세서 PDF를 추가해 호실별 임차인 현황을 보강할 수 있도록 했습니다.
  - 기존 세입자 항목이 부족하거나 없는 경우에도 매각물건명세서에서 추출한 임차인 정보를 병합해 보여주는 방향으로 개선했습니다.
  - 매각물건명세서 파일 선택 후 1초 뒤 자동으로 분석·보강되도록 했습니다.

- **세입자 분석 개선**
  - `배당요구 없는 임차인 기본 평가` 가이드를 추가했습니다.
  - 사용자가 가이드 내용을 수정하고 저장하면 이후 기본값으로 사용되도록 했습니다.
  - 세입자별 `임장 점유` 항목을 추가해 `거주`, `미거주`, `확인필요` 상태를 기록할 수 있습니다.
  - 세입자별 `방크기(㎡)`를 입력해 주변 시세 분석의 유사 월세 비교에 활용할 수 있습니다.
  - 중복 key 오류가 발생하지 않도록 동일한 안내 문구는 중복 제거 후 렌더링합니다.
  - 입주자 이름 수정 중 행이 다시 마운트되지 않도록 세입자 목록 렌더링 key를 이름값에 의존하지 않게 정리했습니다.
  - 세입자 목록은 호실명을 기준으로 지하, 1층, 2층 순서로 정렬해 표시합니다.
  - 세입자 입력칸은 입력 중 즉시 저장하지 않고 Enter 또는 포커스 해제 시 반영해 입력 안정성을 높였습니다.
  - 보증금과 월세가 같은 값으로 들어오면 전세로 보고 월세를 `0`으로 보정합니다.
  - 주택공사/LH/한국토지주택공사 임차인은 녹색 입력 강조로 표시합니다.

- **기본·수동입력 자동 보강**
  - PDF/구조화 문서에서 건폐율, 용적률, 말소기준일 등 기본 정보를 가능한 한 자동 입력하도록 개선했습니다.
  - 세입자·권리 정보에서 말소기준일 후보를 추출해 기본 정보에 반영할 수 있도록 했습니다.
  - 기본정보에서 인근 매각사례 낙찰가·낙찰가율·사례 보증금·월세·메모를 입력하고, 주변 시세 분석의 국토부 매매 평균과 비교할 수 있습니다.
  - 건축물대장 PDF를 `원문/PDF`에서 `건축물대장` 종류로 등록하면 주소, 주용도, 주택/상가 호수, 층별 용도·면적, 대지면적, 연면적, 건폐율, 용적률, 사용승인일 등을 기본정보에 보강합니다.
  - 기본정보에 건축물대장 기준 `층별 구성` 입력란을 추가해 층, 주택/상가 구분, 용도, 면적, 호수를 직접 수정할 수 있습니다.
  - 근린생활시설 포함 물건은 녹색, 평균 주택면적 기준 이하 물건은 주황색, 건물면적 기준 이상 물건은 보라색, 토지가격 기준 이상 물건은 파란색 배지로 표시합니다.
  - 소형 가구 면적 기준(㎡), 대형 건물면적 기준(㎡), 높은 토지가격 기준(만원/㎡)은 기본정보 화면에서 변경할 수 있습니다.

- **주변 월세 및 전세 가격분석**
  - 물건 상세에 `주변 시세 분석` 탭을 추가했습니다.
  - 앱 내부 API가 `MOLIT_API_KEY`로 국토교통부 실거래를 바로 조회해 룸타입별 평균 보증금/월세, 매매 평균, 상세 매물표를 표시합니다.
  - 국토부 단독다가구 전월세 응답의 `계약면적` 필드를 인식하고, 동 단위 결과가 없으면 같은 구의 실거래 결과를 보조로 표시하도록 보강했습니다.
  - 기본 조회 기간은 최근 3개월이며, 일부 국토부 API가 실패해도 성공한 전월세/매매 데이터는 먼저 표시합니다.
  - 같은 동, 인접 동, 면적 유사도, 준공연도 유사도, 최신순 기준으로 시세 목록을 정렬하고 유사 매매 사례 기준 평균을 계산합니다.
  - 건축물대장/룸구성 기준으로 예상 총보증금·총월세를 보조 계산합니다.
  - 국토부 실거래 중 월세가 있는 최근 15건을 방 크기별로 보여주고, 평균·중간값·하한/상한과 건물/방 상태 보정 월세를 계산합니다.
  - 같은 구에 저장된 기존 주변 시세 분석이 있으면 현재 물건의 동·면적·준공연도 기준으로 다시 정렬하고 유사 매매 평균·룸별 요약을 재계산해 재사용할 수 있습니다.
  - 세입자 분석에 입력된 현재 보증금·월세·방크기를 주변 유사 월세 범위와 비교해 `상향 여지`, `적정권`, `높은 편`으로 표시합니다.
  - 최근 월세 실거래나 상세 목록에서 비슷한 거래를 선택해 임대세팅 호실표에 `같은 룸 전체` 또는 `첫 빈칸`으로 보증금·월세를 반영할 수 있습니다.
  - `NEXT_PUBLIC_NAVER_MAP_CLIENT_ID`가 있으면 네이버 지도에 물건과 좌표가 있는 시세 마커를 표시하고, 좌표가 없으면 네이버 지도 검색 링크를 제공합니다.
  - Gemini 해석 JSON이 포함된 경우 시세 강도, 공실 리스크, 추천 룸 구성, 주의사항을 함께 보여줍니다.
  - 해당 물건의 룸 구성에 영향을 주는 월세 실거래를 상단에 우선 표시하고, 전체 매매·전세·월세 상세 및 지도는 아래쪽에서 확인하도록 정리했습니다.

### 주변 시세 분석에 필요한 키

- `MOLIT_API_KEY`: 국토교통부 실거래가 API 수집용입니다.
- `NEXT_PUBLIC_NAVER_MAP_CLIENT_ID`: 앱 화면에서 네이버 지도를 표시할 때 필요합니다. 네이버 지도 JS v3의 `ncpKeyId` 값으로 사용합니다.
- `NAVER_MAP_CLIENT_SECRET`: 주변 시세 조회 시 물건 주소를 좌표로 변환하는 서버 지오코딩에 사용합니다.
- `GEMINI_API_KEY`: 시세 결과를 문장으로 해석하고 위험 코멘트를 만들 때 사용합니다.

- **상세 화면 사용성**
  - 물건 상세 화면은 물건별 마지막으로 보던 탭을 기억합니다.
  - 다시 해당 물건을 열면 매번 기본 탭으로 돌아가지 않고 마지막 탭에서 이어서 볼 수 있습니다.
  - 상세 제목은 `사건번호 · 주소` 형식으로 표시합니다.
  - 기본정보 주소 아래에서 네이버 지도, 카카오맵, 국토부 실거래, 네이버 부동산을 바로 열 수 있습니다.
  - 우선순위는 1~5단계로 관리합니다. 신규 물건은 1단계이며, 정보충실도와 다가구 점수에 따라 3단계까지 시스템 권장이 표시되고, 4~5단계는 사용자가 직접 선호/최우선으로 지정합니다.
  - 물건 목록은 우선순위 단계와 다가구 점수 상위순으로 정렬하며, 건축물대장·매각물건명세서·감정평가서·등기부 등 등록된 원문 자료 배지를 함께 표시합니다.

- **공부하기 화면**
  - `자료조사중` 명칭을 `물건 검색`으로 변경했습니다.
  - 공부하기 페이지의 기본 선택 항목을 `물건 검색`으로 설정했습니다.
  - 상단 메뉴는 유지하면서 선택한 강의 내용만 표시되도록 정리했습니다.

- **데이터 유지와 저장**
  - 기존 로컬 물건 데이터를 유지한 상태에서 기능이 추가되도록 마이그레이션 로직을 보강했습니다.
  - 앱 시작 시 Supabase 데이터는 로컬 데이터를 덮어쓰지 않고 병합합니다.
  - 클라우드 병합, JSON 가져오기, 초기화 직전에는 최근 로컬 상태를 스냅샷 1개로 갱신 보관하고 데이터 화면에서 다운로드·복원할 수 있습니다.
  - 물건 목록이 비어 있는 상태는 자동 Supabase 업로드에서 제외해 빈 데이터가 클라우드에 덮어써지는 일을 줄였습니다.
  - 주변 시세 분석 저장 직전·직후에도 로컬 스냅샷을 남기며, 같은 물건을 병합할 때는 주변 시세 분석값이 있는 쪽을 보존합니다.

## 개발

```bash
npm install
npm run dev
```

브라우저에서 [http://localhost:3000](http://localhost:3000) 을 엽니다.

```bash
npm run build
npm start
```

## 스택

- Next.js (App Router), React, TypeScript  
- Tailwind CSS v4, Zustand(로컬 상태·저장)
